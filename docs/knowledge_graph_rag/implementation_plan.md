# ナレッジグラフ検索機能（GraphRAG）の実装計画

保存されたナレッジグラフに対して、自然言語で質問（例：「山田太郎の新しい届け先はどこですか？」）ができる検索アプリケーション（RAG）を追加します。

## 変更の目的と概要
抽出したナレッジグラフの真価を発揮するため、**「自然言語からCypherクエリを自動生成し（Text2Cypher）、データベースから回答を取得し、さらに自然言語で回答を生成する」**というGraphRAG（グラフベースのRAG）機能を実装します。

## Proposed Changes / 提案する変更内容

### 1. `src/ontology_extractor/neo4j_client.py`
新たに自然言語検索用の処理を追加します。
- `query_kg(question: str) -> str`: ユーザーの質問を受け取り、以下の3ステップで回答を生成するメソッド。
  1. **スキーマ取得**: `fetch_ontology_schema()` で現在のグラフ構造を取得。
  2. **Cypher生成 (LLM)**: スキーマとユーザーの質問をLLMに渡し、Neo4jを検索するためのCypherクエリを生成させる。
  3. **検索と回答生成 (LLM)**: 生成されたCypherをNeo4jで実行し、得られたデータ（JSON）と元の質問を再びLLMに渡し、自然言語で最終的な回答を生成させる。

### 2. `src/ontology_extractor/main.py`
新しいCLIコマンドを追加します。
- `@cli.command()` を用いて、`ask`（または `query`）コマンドを追加。
- 実行例: `uv run ontology-cli ask "山田太郎が変更を希望した届け先はどこですか？"`

> [!WARNING]
> **User Review Required / ユーザー確認事項**
> 
> 1. コマンド名は `ask` （質問する）という名前で進めてよろしいでしょうか？
> 2. ベクトル検索（Embeddingを用いた類似度検索）は一旦入れず、まずは純粋な **Text2Cypher（LLMによるグラフクエリ生成）** のアプローチで実装する形でよろしいでしょうか？（抽出されたグラフの規模が小さく、エンティティのプロパティ検索で十分対応できるため）
