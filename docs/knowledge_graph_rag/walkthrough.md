# ナレッジグラフ検索機能（GraphRAG）実装完了

抽出してNeo4jに格納されたナレッジグラフに対して、自然言語で質問ができる `ask` コマンドを実装しました。この機能は、Text2Cypher（LLMがテキストからCypherクエリを自動生成する技術）を用いたGraphRAGアプローチを採用しています。

## 実装内容
1. `src/ontology_extractor/neo4j_client.py`
   - `query_kg(question)` メソッドを追加しました。
   - **Step 1:** Neo4jから現在のオントロジー（スキーマ）情報を取得。
   - **Step 2:** LLMに対して「このスキーマ構造に従い、ユーザーの質問に答えるためのCypherクエリを生成せよ」と指示し、クエリを生成。
   - **Step 3:** Neo4jでCypherクエリを実行し、結果（JSONデータ）を取得。
   - **Step 4:** 取得したデータと元の質問を再度LLMに渡し、「このデータベースから得られた情報だけを用いて、自然言語でユーザーに回答せよ」と指示して最終的な回答を生成します。

2. `src/ontology_extractor/main.py`
   - コマンドラインツールに `ask` コマンドを追加しました。

## テスト結果
構築したコールセンターのナレッジグラフに対して質問を行った結果、以下のように完璧に動作しました。

**実行コマンド:**
```bash
uv run ontology-cli ask "山田太郎が注文した商品は何ですか？"
```

**裏側のLLMの動き:**
1. Cypherの生成:
   ```cypher
   MATCH (c:Customer {name: '山田太郎'})-[:HAS_ORDERED]->(p:Product) RETURN p.name
   ```
2. DBの検索結果:
   ```json
   [{"p.name": "快眠プレミアムマットレス"}]
   ```
3. 自然言語での回答生成:
   ```text
   --- Answer ---
   山田太郎さんが注文した商品は「快眠プレミアムマットレス」です。
   --------------
   ```

このように、LLM自身がオントロジー（スキーマ）を理解し、自らクエリを組んでグラフデータベースを探索し、適切な回答を生成できるようになりました！
